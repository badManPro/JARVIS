# PRD · Learning Companion（学习伴侣）

## 1. 产品定位
Learning Companion 是一个 **本地运行的桌面学习助手**，面向所有人，但通过 **动态用户画像** 提供个性化学习规划、任务拆解与节奏调整。

它不是通用待办工具，也不是课程平台，而是一个会通过持续问答和使用行为不断理解用户、再基于理解给出学习计划的学习规划产品。

## 2. 核心价值
1. 通过对话建立用户画像
2. 根据画像做个性化学习规划
3. 在执行过程中持续更新画像与计划
4. 本地优先，强调私密性与长期陪伴感

## 3. 目标用户
### 3.1 自学型用户
- 想学新技能，但不会规划
- 需要被拆解、提醒、陪伴

### 3.2 转型型用户
- 想转岗位或补关键技能
- 需要阶段路径和持续调整

### 3.3 兴趣驱动型用户
- 对很多知识有兴趣
- 容易三分钟热度
- 需要更符合个性的规划节奏

## 4. 核心问题
- 学习目标模糊，不会拆解
- 不了解自己的真实学习节奏
- 计划常常脱离个人情况
- 学习工具更多在管任务，不懂“人”

## 5. 产品目标
### 5.1 建立动态用户画像
通过问答采集和后续行为反馈，建立持续更新的用户画像。

### 5.2 生成个性化学习计划
根据画像、目标、时间预算、能力基础输出阶段计划、周计划和日计划。

### 5.3 支持对话式调整
用户可以通过自然语言修改计划、重排任务、调整强度和插入新目标。

## 6. 关键模块
### 6.1 用户画像模块
包含：
- 身份 / 阶段
- 年龄段
- 已掌握技能
- 想学目标
- 时间预算
- 学习偏好
- 性格与行为特征
- 难点与阻力

### 6.2 学习目标模块
用户输入想学什么，并补充：
- 当前基础
- 时间范围
- 目标结果

### 6.3 规划引擎
输入：
- 用户画像
- 学习目标
- 时间预算
- 当前表现

输出：
- 阶段规划
- 周计划
- 日任务
- 调整建议

### 6.4 对话式调整模块
用户可通过聊天说：
- 我最近很累，计划调轻一点
- 我其实已经会一点这个
- 我想临时插入一个目标
- 这周没完成，帮我重排

### 6.5 计划与任务视图
- 首页仪表盘
- 学习路径
- 任务列表
- 复盘建议

### 6.6 用户画像页
- 展示系统当前对用户的理解
- 支持用户手动修正

## 7. 页面结构
建议一级导航：
1. 首页
2. 学习计划
3. 目标
4. 对话
5. 用户画像
6. 复盘
7. 设置

## 8. MVP 范围
### MVP-1 初始问答建档
- 对话式建档
- 生成第一版用户画像

### MVP-2 用户画像页
- 展示系统理解
- 支持修正

### MVP-3 学习目标创建
- 输入想学什么
- 输入目标、基础和周期

### MVP-4 个性化计划生成
- 生成阶段/周/日计划

### MVP-5 对话式调整
- 支持基于原计划调整

### MVP-6 本地桌面客户端壳
- Electron 客户端
- 基本导航与页面骨架

## 9. 成功标准
- 用户可以完成首次建档
- 系统能根据画像生成非模板化的学习计划
- 用户能通过对话修改计划
- 用户能在界面中看到画像、计划与任务的清晰关系

## 10. 产品原则
1. 先理解用户，再给计划
2. 计划必须可执行，而非理想化
3. 用户画像是动态更新的，不是一次性表单
4. 产品优先做“陪伴式规划”，而不是内容平台
5. 界面上坚持 C 端客户端优先：一屏先回答“我现在该做什么”，避免管理后台化
6. 模型能力是可替换能力层，不把产品绑死在单一厂商

## 11. 模型策略
### 11.1 多 Provider 支持
产品从一开始就为多模型 Provider 预留抽象，首批明确考虑：
- GPT / OpenAI
- GLM / 智谱
- Kimi / Moonshot
- DeepSeek

### 11.2 用途路由，而非厂商直连
业务模块描述的是用途：
- 画像提取
- 初版计划生成
- 计划调整
- 复盘总结
- 通用对话

再由设置中的模型策略把这些用途映射到对应 Provider。

### 11.3 设置入口集中管理
Provider、Model、Endpoint、密钥、默认路由策略统一收纳到设置页，不在业务页面分散暴露。

## 12. 当前实现边界
本阶段优先做：
- 产品文档补齐
- 页面骨架与状态层
- 多模型设置入口与前端抽象

本阶段暂不做：
- 真实在线模型调用
- 云同步
- 多人协作
- 复杂自动调度与容灾
