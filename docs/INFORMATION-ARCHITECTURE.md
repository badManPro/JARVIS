## 2. 全局入口

### 全局教练入口
- 作为所有“建档 / 我有变化” 的统一抽屉入口
- 可在任意主页面打开，不单独占用一级导航
- 承担两类主路径：
  - 首次建档：收集主目标、当前水平、时间预算和增强画像，只生成第一版粗版学习路径
  - 变化输入：记录最近变化，标记受影响页面，并把用户带回 `学习路径` 或 `今日` 继续处理

### 上下文复盘入口
- 不再作为独立一级页面
- 今日页在任务标记为 `done / delayed / skipped` 后立即弹出轻量复盘 sheet
- 学习路径页提供阶段复盘入口
- 所有入口都继续写入同一份结构化 reflection 数据，供后续画像和路径建议消费

## 3. 全局界面原则
1. **C 端客户端优先**：页面先帮助个人用户快速决策，不采用后台管理台式密集表格为主的布局。
2. **首屏只给一个主动作**：默认首屏只聚焦当前最该做什么，不一次铺开全部模块。
3. **复杂信息渐进披露**：完整路径依据、历史快照、runtime 细节等信息必须下沉到展开层或高级区。
4. **技术配置集中下沉**：默认设置页只保留高频控制项，Provider / route / observability 集中到高级设置。
5. **对话是能力，不是页面**：自然语言输入通过全局教练入口完成；计划确认与重生成回到业务页面本身完成。

## 4. 今日
### 页面职责
- 作为用户每日打开应用后的唯一主动作入口
- 根据粗版路径和“仅今天有效”的上下文，生成当天细版计划
- 汇总当天聚焦任务、风险提醒和节奏摘要
- 在任务状态变化后直接承接轻量复盘

### 关键区块
1. **今日详细计划 Hero**
   - 今日目标 / 待生成状态
   - 当天投入时长
   - 当天学习窗口
   - 首启时改为建档引导
2. **仅今天有效**
   - 今日可用时长覆盖项
   - 今日学习窗口覆盖项
   - 今日备注
   - `生成今日计划 / 重新生成今日计划`
3. **今日详细计划内容**
   - 学习步骤
   - 资源
   - 练习
   - 今日产出
3. **风险提醒**
   - 当前最重要风险
   - 风险原因
   - 建议动作
4. **当前节奏**
   - 本周完成率
   - 连续天数
   - 最近复盘摘要
5. **粗版路径进度**
   - 当前粗版任务
   - 开始 / 完成 / 延后 / 跳过
   - 复盘承接

### 核心交互
- 保存当天覆盖项
- 手动生成或重生成今日详细计划
- 标记完成 / 延后 / 跳过
- 打开教练入口描述变化
- 跳转到学习路径
- 在状态变化后立即填写轻量复盘

## 5. 学习路径
### 页面职责
- 作为长期粗版计划的固定查看入口
- 展示当前主目标、周里程碑、当前关键节点和重要检查点
- 允许在多目标之间切换当前主线
- 把复杂解释信息下沉到渐进披露层

### 关键区块
1. **当前粗版路径 Hero**
   - 当前主目标
   - 当前路径摘要
   - `重新生成粗版计划`
   - 发起调整
   - 阶段复盘
2. **周里程碑**
   - 默认按周展示粗版路径
   - 当前周里程碑 / 下一节点 / 已完成节点
3. **当前关键节点**
   - 当前周目标
   - 验收结果
   - 当前阶段
4. **重要检查点**
   - 阶段性 outcome
   - 当前进度
5. **最近任务**
   - 作为辅助信息保留
6. **切换主目标**
   - 当前激活目标
   - 多目标切换
5. **高级内容**
   - 完整路径依据
   - 历史快照

### 核心交互
- 切换主目标
- 查看粗版周里程碑和关键节点
- 手动重生成粗版计划
- 发起路径调整
- 记录阶段复盘
- 在展开层查看完整路径依据与历史快照

## 6. 学习档案
### 页面职责
- 呈现系统当前对用户的学习画像
- 允许用户直接修正背景、节奏、性格和反馈偏好
- 解释这些信息如何影响任务拆解和提醒方式

### 关键区块
1. **学习人物画像 Hero**
   - 当前画像定位
   - 系统使用原则说明
2. **学习背景**
   - 当前水平
   - 时间预算
   - 学习窗口
3. **人物特征**
   - 年龄阶段
   - MBTI
   - 性格关键词
   - 压力偏好
4. **系统如何使用**
   - 画像如何影响任务拆解
   - 画像如何影响反馈风格
5. **编辑档案**
   - 手动编辑增强画像字段
   - 保存 / 取消

### 核心交互
- 展开和收起编辑
- 手动修正画像字段
- 让系统基于档案调整后续节奏与提醒方式

## 7. 设置
### 页面职责
- 管理高频基础偏好和 Codex 连接
- 通过高级设置承接技术配置
- 保持业务页面不暴露 runtime / provider 噪音

### 关键区块
1. **基础设置 Hero**
   - 分层原则说明
2. **基础偏好**
   - 主题
   - 启动页
3. **连接 Codex**
   - 当前连接状态
   - 连接 Codex
   - 检查连接
   - 断开连接
   - 设备码回退
4. **高级设置**
   - 用途路由概览
   - AI 运行时摘要
   - 观测摘要
   - Provider 编辑与健康检查

### 核心交互
- 保存基础设置
- 连接 / 检查 / 断开 Codex
- 在高级设置中调整 route、Provider 和健康检查
- 查看最小 observability 摘要

## 8. 已移除的旧一级结构
- 旧版 `首页 / 学习计划 / 目标 / 对话 / 用户画像 / 复盘 / 设置` 七页结构不再使用
- 目标能力已并入学习路径和教练入口
- 对话能力已并入全局教练抽屉
- 复盘能力已并入任务状态变化后的上下文动作
- 教练入口中的 accept / reject / apply 预览控制台不再保留
