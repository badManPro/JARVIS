## 0. 全局界面原则
1. **C 端客户端优先**：页面先帮助个人用户快速决策，不采用后台管理台式密集表格为主的布局。
2. **首屏给动作**：每个一级页面的首屏都要有“我接下来能做什么”。
3. **上下文要连起来**：画像、目标、计划、对话、复盘之间要能互相解释，不是孤岛页面。
4. **模型配置集中到设置**：业务页面不散落 Provider 细节，统一由设置页承接。

## 2. 首页
### 页面职责
- 作为用户每日打开应用后的总览入口
- 汇总今日学习状态、计划进度、风险提醒与建议动作
- 将“画像 → 目标 → 计划 → 执行 → 复盘”的链路压缩成可快速扫描的看板

### 关键区块
1. **顶部欢迎区**
   - 当前日期
   - 用户昵称
   - 今日一句学习提示
2. **今日聚焦卡片**
   - 今日最重要任务
   - 预计投入时长
   - 当前计划阶段
3. **学习进度总览**
   - 本周完成率
   - 连续学习天数
   - 当前目标推进状态
4. **待处理提醒**
   - 延迟任务
   - 即将到期里程碑
   - 需要用户确认的计划调整
5. **快捷入口**
   - 开始今日学习
   - 调整计划
   - 新建目标
   - 发起对话
6. **近期复盘摘要**
   - 最近一次复盘结论
   - 当前阻碍关键词

### 核心交互
- 从首页进入当日计划详情
- 对提醒项执行“重排/延后/标记完成”
- 一键跳转到对话页继续沟通

## 3. 学习计划
### 页面职责
- 展示系统基于目标与画像生成的学习路径
- 支持按阶段、周、日查看与调整计划
- 让用户理解“为什么这么安排”而不只是看到任务列表

### 关键区块
1. **计划摘要栏**
   - 当前激活计划
   - 对应目标
   - 计划周期
   - 总时长预算
2. **视图切换**
   - 阶段视图
   - 周视图
   - 日视图
3. **路径时间线**
   - 阶段名称
   - 阶段目标
   - 关键里程碑
4. **任务清单区**
   - 任务名称
   - 预计时长
   - 状态
   - 优先级
5. **计划依据说明**
   - 基于哪些画像特征做出当前安排
   - 当前强度/节奏说明
6. **调整操作区**
   - 重排任务
   - 降低/提高强度
   - 插入临时任务
   - 标记计划失效并重新生成

### 核心交互
- 在不同粒度计划间切换
- 对单个任务进行完成、跳过、延后
- 发起“基于当前进度重排”的对话动作

## 4. 目标
### 页面职责
- 管理用户所有学习目标
- 明确每个目标的起点、结果定义、时间边界与优先级
- 支撑多目标并行或切换场景

### 关键区块
1. **目标列表**
   - 目标名称
   - 状态（进行中/已暂停/已完成）
   - 优先级
2. **目标详情卡**
   - 想达到的结果
   - 当前基础
   - 目标周期
   - 成功判定标准
3. **目标关系与排序**
   - 主目标/次目标
   - 是否共享时间预算
4. **创建/编辑表单**
   - 目标名称
   - 背景动机
   - 现有基础
   - 可投入时间
   - 截止时间
5. **与计划联动提示**
   - 是否已生成计划
   - 最近一次计划更新时间
   - 当前计划覆盖度

### 核心交互
- 创建新目标
- 调整优先级
- 将目标加入现有计划或重新规划

## 5. 对话
### 页面职责
- 作为产品的自然语言交互中心
- 承担初次建档、计划调整、节奏反馈和追问澄清
- 将用户话语转译为画像或计划的结构化变更

### 关键区块
1. **会话头部**
   - 当前会话主题
   - 所属目标/计划
   - 上次更新时间
2. **消息流**
   - 用户消息
   - 系统建议
   - 结构化变更提示
3. **建议操作卡**
   - 采用调整建议
   - 仅更新画像
   - 稍后处理
4. **输入区**
   - 文本输入
   - 快捷问题模板
5. **上下文侧栏**
   - 当前引用的目标
   - 当前引用的计划
   - 相关画像标签

### 核心交互
- 通过自然语言表达变化：没时间、太难、目标变了
- 预览系统要更新的内容，再确认落库
- 从对话直接跳到相关页面

## 6. 用户画像
### 页面职责
- 呈现系统当前对用户的理解模型
- 支持用户校正系统理解，形成更可靠的规划输入
- 让“画像如何影响计划”可解释

### 关键区块
1. **画像总览卡**
   - 当前学习身份
   - 目标倾向
   - 时间预算
   - 学习节奏偏好
2. **基础信息区**
   - 阶段/背景
   - 已掌握技能
   - 当前水平判断
3. **行为与偏好区**
   - 偏好学习方式
   - 容易中断的原因
   - 最佳学习时段
4. **风险与阻力区**
   - 拖延触发因素
   - 容易失败的场景
   - 压力阈值
5. **画像变更记录**
   - 最近更新内容
   - 来源（对话/手动编辑/系统推断）
6. **对计划影响说明**
   - 哪些画像特征影响了当前计划强度和节奏

### 核心交互
- 手动修正画像字段
- 查看画像更新历史
- 从画像字段跳到相关计划依据说明

---

## 7. 复盘
### 页面职责
- 汇总学习执行结果与趋势
- 帮助用户回顾有效方法、阻碍因素和后续调整建议
- 为下一轮画像更新与计划重排提供依据

### 关键区块
1. **周期切换**
   - 日复盘
   - 周复盘
   - 阶段复盘
2. **完成情况统计**
   - 完成任务数
   - 实际投入时长
   - 偏差情况
3. **问题归因区**
   - 未完成原因
   - 任务难度是否匹配
   - 时间分配是否合理
4. **系统建议区**
   - 调整节奏
   - 缩减范围
   - 前置补基础
5. **用户自评区**
   - 心情/压力
   - 自信度
   - 主观成就感
6. **后续动作区**
   - 生成下周期建议
   - 同步更新画像
   - 重排计划

### 核心交互
- 填写结构化复盘
- 对系统建议进行采纳/忽略
- 将复盘结果推送到计划与画像模块

## 8. 设置
### 页面职责
- 管理应用层配置、模型接入、本地数据和偏好项
- 保持桌面应用的可控性与本地优先体验

### 关键区块
1. **账号与本地身份**
   - 昵称
   - 本地设备标识
   - 数据目录说明
2. **AI 配置**
   - 模型服务提供方（GPT / GLM / Kimi / DeepSeek / Custom）
   - API Key / 本地模型地址 / Endpoint
   - 按用途分配的默认路由策略
   - Provider 健康状态与启用开关
3. **应用偏好**
   - 主题模式
   - 启动页
   - 通知开关
4. **数据管理**
   - 导出数据
   - 导入备份
   - 清理缓存
5. **隐私与安全**
   - 本地存储说明
   - 诊断日志开关
6. **关于**
   - 版本号
   - 路线图链接
   - 文档入口

### 核心交互
- 修改 AI 与外观设置
- 导入导出本地数据
- 查看版本与项目文档

---

## 9. 页面关系说明
- 首页聚合学习计划、目标、复盘的关键状态
- 学习计划由目标与用户画像共同驱动
- 对话可修改目标、画像和计划，但重要变更需确认
- 复盘沉淀执行反馈，并回流到画像与计划
- 设置独立于业务页面，但为 AI 能力和本地数据提供底层支持
