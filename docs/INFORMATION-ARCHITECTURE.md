## 2. 全局入口

### 全局教练入口
- 作为所有“对话 / 建档 / 我有变化 / 建议确认”的统一抽屉入口
- 可在任意主页面打开，不单独占用一级导航
- 承担两类主路径：
  - 首次建档：收集主目标、当前水平、时间预算和增强画像，直接生成第一版学习路径
  - 变化输入：记录最近变化，生成画像建议或路径调整建议，并在抽屉内完成 accept / reject / apply 闭环

### 上下文复盘入口
- 不再作为独立一级页面
- 今日页在任务标记为 `done / delayed / skipped` 后立即弹出轻量复盘 sheet
- 学习路径页提供阶段复盘入口
- 所有入口都继续写入同一份结构化 reflection 数据，供后续画像和路径建议消费

## 3. 全局界面原则
1. **C 端客户端优先**：页面先帮助个人用户快速决策，不采用后台管理台式密集表格为主的布局。
2. **首屏只给一个主动作**：默认首屏只聚焦当前最该做什么，不一次铺开全部模块。
3. **复杂信息渐进披露**：完整路径依据、历史快照、runtime 细节等信息必须下沉到展开层或高级区。
4. **技术配置集中下沉**：默认设置页只保留高频控制项，Provider / route / observability 集中到高级设置。
5. **对话是能力，不是页面**：自然语言输入和建议确认通过全局教练入口完成，而不是重新回到独立 tab。

## 4. 今日
### 页面职责
- 作为用户每日打开应用后的唯一主动作入口
- 汇总当前聚焦任务、风险提醒和节奏摘要
- 在任务状态变化后直接承接轻量复盘

### 关键区块
1. **今日主任务 Hero**
   - 当前最重要任务
   - 预计投入时长
   - 当前阶段
   - 首启时改为建档引导
2. **现在要做什么**
   - 当前唯一聚焦任务
   - 开始 / 完成 / 延后
   - “我有变化”入口
3. **风险提醒**
   - 当前最重要风险
   - 风险原因
   - 建议动作
4. **当前节奏**
   - 本周完成率
   - 连续天数
   - 最近复盘摘要

### 核心交互
- 开始当前任务
- 标记完成 / 延后 / 跳过
- 打开教练入口描述变化
- 跳转到学习路径
- 在状态变化后立即填写轻量复盘

## 5. 学习路径
### 页面职责
- 展示当前主目标、当前阶段和最近任务
- 允许在多目标之间切换当前主线
- 把复杂解释信息下沉到渐进披露层

### 关键区块
1. **当前主线 Hero**
   - 当前主目标
   - 当前路径摘要
   - 发起调整
   - 阶段复盘
2. **切换主目标**
   - 当前激活目标
   - 多目标切换
3. **最近任务**
   - 最近 3-5 个任务
   - 时长
   - 状态
4. **当前阶段**
   - 阶段标题
   - 阶段目标
   - 当前进度
5. **高级内容**
   - 完整路径依据
   - 历史快照

### 核心交互
- 切换主目标
- 查看当前阶段和最近任务
- 发起路径调整
- 记录阶段复盘
- 在展开层查看完整路径依据与历史快照

## 6. 学习档案
### 页面职责
- 呈现系统当前对用户的学习画像
- 允许用户直接修正背景、节奏、性格和反馈偏好
- 解释这些信息如何影响任务拆解和提醒方式

### 关键区块
1. **学习人物画像 Hero**
   - 当前画像定位
   - 系统使用原则说明
2. **学习背景**
   - 当前水平
   - 时间预算
   - 学习窗口
3. **人物特征**
   - 年龄阶段
   - MBTI
   - 性格关键词
   - 压力偏好
4. **系统如何使用**
   - 画像如何影响任务拆解
   - 画像如何影响反馈风格
5. **编辑档案**
   - 手动编辑增强画像字段
   - 保存 / 取消

### 核心交互
- 展开和收起编辑
- 手动修正画像字段
- 让系统基于档案调整后续节奏与提醒方式

## 7. 设置
### 页面职责
- 管理高频基础偏好和 Codex 连接
- 通过高级设置承接技术配置
- 保持业务页面不暴露 runtime / provider 噪音

### 关键区块
1. **基础设置 Hero**
   - 分层原则说明
2. **基础偏好**
   - 主题
   - 启动页
3. **连接 Codex**
   - 当前连接状态
   - 连接 Codex
   - 检查连接
   - 断开连接
   - 设备码回退
4. **高级设置**
   - 用途路由概览
   - AI 运行时摘要
   - 观测摘要
   - Provider 编辑与健康检查

### 核心交互
- 保存基础设置
- 连接 / 检查 / 断开 Codex
- 在高级设置中调整 route、Provider 和健康检查
- 查看最小 observability 摘要

## 8. 已移除的旧一级结构
- 旧版 `首页 / 学习计划 / 目标 / 对话 / 用户画像 / 复盘 / 设置` 七页结构不再使用
- 目标能力已并入学习路径和教练入口
- 对话能力已并入全局教练抽屉
- 复盘能力已并入任务状态变化后的上下文动作
