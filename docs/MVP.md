# MVP · Learning Companion

## MVP 目标
先做一个能真正跑通“理解用户 → 生成计划 → 对话调整”的本地桌面客户端。

## 范围
### 0. 产品体验前提
- 坚持 C 端客户端优先，不把 MVP 做成后台管理面板
- 设置页提供多模型 Provider 的统一入口，但不要求首版打通真实请求

### 1. 初始建档
- 通过问答建立第一版用户画像

### 2. 用户画像页
- 查看系统理解
- 手动修改关键字段

### 3. 学习目标创建
- 输入目标、基础、周期

### 4. 计划生成
- 输出阶段计划
- 输出本周与今日任务

### 5. 对话式调整
- 根据对话更新计划

### 6. 本地客户端基础界面
- 首页
- 计划页
- 对话页
- 用户画像页
- 设置页

## 不在第一版范围
- 多人协作
- 云同步
- 内容课程平台
- 本地模型复杂推理链
- 高级统计分析
